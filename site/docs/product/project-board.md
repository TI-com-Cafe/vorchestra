# Project board

The project board changes the app from venv-first to project-first.

## What it shows

- Project root.
- Associated environments.
- Best environment health.
- Detected manifests and lockfiles.
- Next-best actions.

## Common actions

- Scan project.
- Open Studio.
- Run diagnostics.
- Repair environment.
- Sync from project manifests.
- Use uv-native workflows such as `uv sync`, `uv lock`, `uv add`, `uv remove`, and `uv run` when supported.
